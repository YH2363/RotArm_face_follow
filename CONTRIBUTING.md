# 贡献指南 (Contributing Guide) — 人脸追踪机械臂

感谢你对我们项目的关注！这是一个结合了**计算机视觉**、**PID 控制算法**与**嵌入式硬件**的综合性项目。我们非常欢迎任何形式的贡献，无论是算法优化、硬件适配，还是文档改进。

**特别提醒**：本项目涉及真实物理硬件（机械臂）的运动。在贡献代码或测试时，请务必遵循本指南中的[安全须知](#-安全第一)章节，确保设备和人员安全。

---

## ⚠️ 安全第一

在操作机械臂之前，请务必遵守以下安全准则：

1.  **限位保护**：代码中已设置了舵机角度限位（`PAN_LIMITS`、`TILT_LIMITS`），**请勿随意修改**至超出物理极限的范围，否则可能烧毁舵机。
2.  **手扶备用**：首次运行或修改 PID 参数后，建议**手持急停开关**或用手指轻扶舵机臂，防止因参数不当导致剧烈抖动或撞机。
3.  **串口占用**：确保 `Arm_Lib` 串口未被其他程序占用，否则舵机指令将无法发送，可能导致机械臂静止而 PID 积分饱和（Integral Windup），引发异常大幅转动。
4.  **环境光线**：人脸检测受光线影响较大，请在稳定的光照环境下测试，避免强逆光导致检测失效。

## 🐛 报告 Bug 或提出建议

如果你在使用过程中遇到了问题，或有改进想法，请先查阅 [Issues](https://github.com/YH2363/RotArm_face_follow/issues) 查看是否已有类似讨论。若没有，请**创建一个新的 Issue**，并尽量提供以下信息：

- **问题描述**：清晰描述现象（例如：追踪时舵机剧烈抖动 / 人脸检测不到 / 程序启动报错）。
- **复现步骤**：详细说明从启动到出现问题你执行了哪些操作。
- **预期行为 vs 实际行为**：你期望发生什么？实际发生了什么？
- **环境信息**：
  - 操作系统（Ubuntu / Windows / 树莓派 OS）
  - Python 版本
  - OpenCV 版本
  - 是否在 Jupyter Notebook 中运行
  - 摄像头型号和分辨率
- **日志/截图**：如果有错误堆栈信息或画面异常截图，请附上。

## 🧑‍💻 贡献代码（Pull Request）

### 1. Fork 本仓库
点击 GitHub 右上角的 **Fork** 按钮，将项目复制到你自己的账号下。

### 2. 克隆你的 Fork 到本地
```bash
git clone https://github.com/你的用户名/face-tracking-robot.git
cd face-tracking-robot
```

### 3. 创建特性分支
请基于 `main` 分支创建新分支，分支名应能反映你的工作内容：
```bash
git checkout -b feature/你的功能
# 例如：feature/optimize-pid-params
# 例如：fix/imu-compensation
# 例如：docs/update-readme
```

### 4. 搭建开发环境
本项目同时支持 `.py` 脚本运行和 Jupyter Notebook 运行，请确保依赖完整：
```bash
pip install opencv-python numpy ipywidgets
```
> **硬件依赖**：`Arm_Lib` 需要根据你的机械臂型号自行安装，请参考硬件厂商的 SDK 文档。

### 5. 代码规范与注释
为了让代码更易维护，请遵守以下规范：

- **Python 风格**：严格遵循 **PEP 8**。建议使用 `flake8` 或 `black` 检查格式。
- **文档字符串**：新增的函数或类必须包含 Google 风格或 NumPy 风格的 docstring，描述参数、返回值和功能。
- **配置集中化**：所有可调参数（如 `PAN_KP`、`DEAD_ZONE` 等）请集中在文件开头的“集中配置”区域，不要硬编码在函数逻辑中。
- **类型注解**：鼓励使用 Python 类型注解（如 `def update(self, error: float, now: float) -> float:`），提升代码可读性。

### 6. 逻辑修改与测试（关键步骤）

根据你修改的内容，进行针对性的手动测试：

- **修改了 PID 参数或控制逻辑**：
  - 建议先在**无机械臂连接**的环境下运行 `face_follow.py`，观察终端打印的角度变化是否平滑合理。
  - 确认无误后，连接机械臂，**手扶舵机**进行实机测试，确保运动方向正确且无卡顿。
- **修改了人脸检测逻辑**：
  - 测试不同光照条件下（正常光、侧光）的检测鲁棒性。
  - 测试不同距离（近处 30cm，远处 2m）的追踪表现。
- **修改了界面或显示逻辑**：
  - 确认 `ipywidgets` 画面显示正常，FPS 计数准确，Exit 按钮响应灵敏。

**测试完成后**，请在 PR 描述中简单说明你的测试环境与结果。

### 7. 提交与推送
提交信息请使用清晰、描述性的语言：
```bash
git add .
git commit -m "优化: 增加俯仰方向加速度限幅，减少末端抖动"
# 提交信息格式建议：<类型>: <简短描述>
# 类型如：优化 / 修复 / 新增 / 文档
git push origin feature/你的功能
```

### 8. 创建 Pull Request (PR)
- 在你 Fork 的仓库页面点击 **Compare & pull request**。
- 在 PR 描述中写明：
  - **本次修改解决了什么问题**（关联 Issue 编号）。
  - **修改的核心逻辑/算法**。
  - **测试结果**（附截图或动图更佳）。
- 等待维护者审阅，我们会尽快回复。

## 🔧 在 Jupyter Notebook 中开发时的特别提示

本项目原始代码设计为在 `.py` 中运行，如果你习惯在 `.ipynb` 中迭代开发，请注意：

1.  **重复运行单元格**：如果多次重复运行初始化单元格，会导致 `Arm_Lib` 重复实例化，可能引发串口冲突。建议在修改后**重启 Kernel** 再运行。
2.  **线程管理**：`camera` 函数运行在后台线程中，若 Notebook 卡死或退出异常，需手动重启 Kernel 释放摄像头资源。
3.  **变量覆盖**：修改配置变量（如 `PAN_KP`）后，需重新实例化 `FaceTrack` 对象才能生效。

## 🧪 关于自动化测试

目前项目尚未引入自动化测试框架（因涉及真实硬件，单元测试覆盖较难）。但我们鼓励贡献者为**纯逻辑函数**（如 `clamp`、`slew`、`face_filter`）编写单元测试（使用 `pytest`），以提高代码的可靠性。
